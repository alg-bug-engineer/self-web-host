# 项目代号：Project Cheese-Fish (芝士吃鱼)

**目标：** 构建一个兼具“亲和力（漫画）”与“专业度（技术）”的个人 IP 门户站。
**核心隐喻：** 猫代表“遇到困难的提问者/人类”，机器人代表“解决问题的技术/AI”。

---

## 一、 视觉识别系统 (VIS) - 给 UI 设计/前端

基于封面图提取的配色方案，保持品牌一致性：

### 1. 配色方案 (Color Palette)

* **主色调 (Brand Blue):** `#5DE2FF` (取自机器人面部光晕)
* *用途：* 按钮、链接、高亮文字、科技侧（CI 网站）的主题色。


* **辅助暖色 (Sunset Orange):** `#FFD066` 到 `#FF9F43` 的渐变 (取自背景夕阳)
* *用途：* 漫画侧（Manga 网站）的主题色、Hover 效果、强调背景。


* **中性色 (Cat Black & White):**
* `#2D3436` (猫身条纹黑 - 用于正文)
* `#F5F6FA` (机器人机身白 - 用于网页背景，不要纯白，要带一点点冷灰)



### 2. 字体与图标 (Typography & Icons)

* **字体：** 推荐使用系统字体栈，但标题需加粗。
* *标题：* `PingFang SC`, `Microsoft YaHei`, `Inter`, `Bold` (为了呼应漫画，标题可以选稍微圆润一点的字重)。
* *代码/技术部分：* `JetBrains Mono` 或 `Fira Code` (体现极客感)。


* **吉祥物应用：**
* **猫** = 代表“问题”或“阅读”。
* **机器人** = 代表“工具”或“解决方案”。



---

## 二、 页面布局规范 (Page Layout) - 给前端开发

采用 **“单页流 + 模态框/跳转”** 结构。重点在于**首屏**和**分流区**。

### 1. 导航栏 (Navbar)

* **左侧：** Logo（提取封面中的猫头+机器人头 ICON） + 文字 “AI 知识点 | 芝士AI吃鱼”。
* **右侧：** 简洁链接：`漫画科普` | `效率工具` | `深度博客` | `关于我`。
* **交互：** 滚动时背景半透明高斯模糊 (Backdrop-blur)。

### 2. 首屏 (Hero Section) - **核心开发难点**

**设计意图：** 还原封面的对话感。

* **布局：** 左右分栏（移动端上下）。
* **左侧（文案）：**
* H1: **“把 AI 天书，讲成人话。”** (大字，使用黑色)
* H2: *“左手画漫画，右手写代码。我是 [你的名字]，带你从 0 到 1 玩转 AI。”* (灰色副标)
* **动态打字机效果 (Typewriter Effect)：** 在副标题下方，显示一行变化的代码/问题：
* `> 用户：Transformer 是什么？` (猫的图标)
* `> AI：让我画个图给你看...` (机器人的图标)




* **右侧（视觉）：**
* 放置你的 **IP 封面原图**（去底 PNG 最好，或者带圆形遮罩）。
* **微动效：** 机器人的天线可以微微发光（CSS `box-shadow` 动画），猫的尾巴可以做轻微摆动（CSS `rotate` 动画）。



### 3. 分流区块 (The Fork) - **最重要的转化区**

这里要用 **Bento Grid (便当盒)** 风格，把两个子站做成两张大卡片。

* **卡片 A：[漫画 AI 知识点] (指向 manga 子站)**
* **背景色：** 浅黄色淡入 (辅助暖色)。
* **ICON：** 那只**高冷的猫**。
* **标题：** “我是小白，我想看漫画”
* **描述：** 拒绝枯燥公式，用故事讲透大模型原理。
* **按钮：** `开始阅读 ->` (实心黑色按钮，圆角)。


* **卡片 B：[CI 效率工具站] (指向 ci 子站)**
* **背景色：** 浅蓝色淡入 (主色调)。
* **ICON：** 那个**发光的机器人**。
* **标题：** “我是极客，我要提效”
* **描述：** 自动化工作流，只管输入，AI 帮你整理。
* **按钮：** `使用工具 ->` (空心蓝色描边按钮)。



### 4. 资源与文章 (Resources & Blog)

* 采用**列表布局**而非卡片（显得更专业、信息密度更高）。
* 每行左侧显示标签：`[深度]` 或 `[资源]`。
* **创新点：** 在每一条资源的推荐语旁边，放一个小的**猫头表情**（表示吐槽）或**机器人头表情**（表示推荐）。
* *例：* `DeepSeek R1 论文精读` - 🤖 机器人说：这篇必须读，不仅是因为它开源...



### 5. 页脚 (Footer)

* 背景色使用深色（封面中海水的颜色变深）。
* 内容：
* “Copyright © 2025 芝士AI吃鱼”
* 友情链接（你的 CSDN、知乎、Github）。
* **彩蛋：** 底部加一句小字：“本站由 🐱 监督，🤖 生成。”

## 三、 交互体验 (UX) - 给开发者的 Note

1. **加载状态 (Loading)：**
* 不要用传统的转圈圈。
* 用封面里的**机器人脸 (`> w <`)** 做 Loading 动画，眼睛眨一眨。


2. **鼠标 Hover 效果：**
* 鼠标移动到“漫画卡片”上时，光标变成猫爪 🐾。
* 鼠标移动到“工具卡片”上时，光标变成代码光标 `|` 或扳手 🔧。


3. **移动端适配：**
* 封面图在手机上可能会占据太多垂直空间，建议在手机端将 Hero Section 的图片缩小或作为背景虚化处理，优先展示“两个分流卡片”。



---

## 四、 开发技术栈建议

为了轻量化且利于 SEO（因为你要做 IP）：

* **框架：** Next.js (React) 或 Astro (速度极快，适合内容站)。
* **样式：** Tailwind CSS (能最快实现上述的颜色和圆角配置)。
* **部署：** Vercel 或 Cloudflare Pages (自带 CDN，速度快)。

---

## 五、 给研发的一句话需求总结

> “老铁，这个网站的核心是**反差萌**。UI 框架要干净、现代（像 Linear 或 Vercel 那种风格），但是**内容和插图**要用我们的猫和机器人。首屏一定要把那张封面图用好，那是我们的门面。切记不要做成传统的企业官网，要做成一个**有趣的个人实验室**。”
